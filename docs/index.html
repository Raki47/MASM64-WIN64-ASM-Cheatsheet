WIN64 Assembly Cheatsheet

MASM64 — work in progress

What is this?

Every program on your computer — Chrome, Photoshop, your games — eventually becomes raw instructions sent directly to the CPU. Most programming languages hide this from you behind layers of translation. Assembly is what's at the bottom. You're writing the actual instructions the chip executes, one step at a time, with nothing in between.

MASM64 is the tool for writing that kind of code on 64-bit Windows. This page is working notes for someone learning how to talk to the processor directly.

The registers listed here are the CPU's scratchpad — a handful of ultra-fast slots built into the chip itself. Every calculation, memory read, and function call runs through them. There are only about 16, and each one has a job.

General Purpose Registers

64-bit Registers

RAX Accumulator register Used for arrithmatic operations and as a return register.
RBX Base register Used for base addressing.
RCX Counter register Used for loop counters and as the first argument In function calls.
RDX Data register Used for I/O operations and as the second argument In function calls.
RSI Source Index Typically used for string and array memory operations.
RDI Destination Index Also used for string and array operations, RSI and RDI also often used as source and destination pointers.
RSP Stack Pointer Points to the top of the stack.
RBP Base Pointer Often ommited by modern compilers, freeing It to be used as a general purpose register.
RIP Instruction Pointer Points to the next Instruction to be executed. (64-bit exclusive register)
R8–R15 Extended Registers Additional general purpose registers for extended operations and functional arguments.

32-bit Registers

These Instructions can be translated for 32 bit ASM or using 32 bit functions/code In MASM64 by replacing the "R" with an "E", for example:

RAX = EAX, RBX = EBX, ect.

The exception to this is the R8-R15 registers, with these instead of replacing the "R" you add a "D" at the end, for example:

R8 = R8D, R9 = R9D, R10 = R10D, ect.

16-bit Registers

This follows a similar rule to translating 64 bit to 32 bit, except instead of swapping the "R" you remove the first letter completely, for example:

RAX = AX, RBX = BX, ect.

With 16 bit, R8-R15 now become R8W-R15W.

Windows x64 Calling Convention

When your code calls a function — whether it's one you wrote or a Windows API function — both sides need to agree on how to hand data back and forth. That agreement is called the calling convention. On Windows x64 there is only one, and it's strict.

Arguments are loaded into specific registers before the call, in order. If you're calling a function that takes three inputs, the first goes in RCX, the second in RDX, the third in R8. The function knows exactly where to look.

Argument Registers (in order)

RCX 1st argument The first value passed into a function call.
RDX 2nd argument The second value passed into a function call.
R8 3rd argument The third value passed into a function call.
R9 4th argument The fourth value passed into a function call.
Stack 5th argument onwards Any arguments beyond the fourth are pushed onto the stack instead.

Return Value

When a function finishes and hands a value back to the caller, that value is placed in RAX. If you call a function and want to use the result, RAX is where you look after the call returns.

Volatile vs Non-Volatile Registers

Some registers are considered "volatile" — the function you call is allowed to overwrite them without restoring them. If you need a value to survive a function call, don't leave it in a volatile register.

Volatile (may be changed by a call): RAX, RCX, RDX, R8, R9, R10, R11

Non-volatile registers must be saved and restored by any function that uses them, so their values are safe across calls.

Non-volatile (safe across calls): RBX, RBP, RDI, RSI, RSP, R12–R15

Common Instructions

Instructions are the actual commands you give the CPU. Each one does a single, specific thing. Below are the ones you'll use constantly when starting out.

MOV Move / Copy Copies a value from one place to another. The source is unchanged. This is the most used instruction in assembly.
ADD Add Adds two values together and stores the result in the destination register.
SUB Subtract Subtracts the second value from the first and stores the result in the destination register.
INC Increment Adds 1 to a register. Shorter than writing ADD RAX, 1.
DEC Decrement Subtracts 1 from a register.
PUSH Push to stack Places a value on top of the stack and moves RSP down to account for it.
POP Pop from stack Takes the top value off the stack, puts it in the specified register, and moves RSP back up.
CMP Compare Compares two values by subtracting them, but throws the result away. It only sets the status flags, which conditional jumps then read.
JMP Jump Unconditionally jumps to a label in your code. Execution continues from there.
CALL Call a procedure Jumps to a function or procedure. Before jumping, it saves the return address on the stack so RET knows where to come back to.
RET Return Ends the current procedure and jumps back to wherever CALL was made from.
XOR Exclusive OR Performs a bitwise XOR. Commonly used to zero out a register — XOR RAX, RAX sets RAX to 0 faster than MOV RAX, 0.

Conditional Jumps

A conditional jump only jumps if a certain condition is true. They're almost always used right after a CMP instruction. CMP sets flags behind the scenes, and then the jump reads those flags to decide whether to go or not.

Think of it like an if statement: CMP is the condition check, the jump is the branch.

JE Jump if Equal Jumps if the two compared values were equal. Also written JZ (jump if zero).
JNE Jump if Not Equal Jumps if the two values were not equal. Also written JNZ (jump if not zero).
JL Jump if Less Than Jumps if the first value was less than the second. For signed numbers.
JG Jump if Greater Than Jumps if the first value was greater than the second. For signed numbers.
JLE Jump if Less or Equal Jumps if the first value was less than or equal to the second. For signed numbers.
JGE Jump if Greater or Equal Jumps if the first value was greater than or equal to the second. For signed numbers.
JA Jump if Above Like JG but for unsigned numbers.
JB Jump if Below Like JL but for unsigned numbers.

Signed vs Unsigned

Signed numbers can be negative (they use one bit to track the sign). Unsigned numbers are always positive and use all bits for the value. Use JL/JG for signed comparisons, JA/JB for unsigned. Getting this wrong is a common source of bugs.

Status Flags (RFLAGS)

The CPU keeps a special register called RFLAGS that tracks the outcome of operations. You don't usually set these manually — instructions like CMP, ADD, and SUB set them automatically as a side effect. Conditional jumps then read them to decide what to do.

ZF Zero Flag Set to 1 when the result of an operation is zero. CMP sets this when two values are equal.
CF Carry Flag Set when an unsigned operation overflows or borrows past its bit limit. Used by JA and JB.
SF Sign Flag Set to 1 when the result is negative (the top bit is 1). Mirrors the sign bit of the result.
OF Overflow Flag Set when a signed operation produces a result too large or too small to fit. Used by JL and JG.
